# Prism Vault Role Review

## Reviewed roles

| Role | Hard question | Finding | Decision |
|---|---|---|---|
| Aha Graph Editor | What does the team realize, and how do they know it is earned? | Current chain is fair but too visually similar between zones. | Add distinct frames/palette per beat and make the beam state the dependency graph. |
| Trailer Flow Cartographer | Where do bodies stand, turn, wait, search, and get out? | Existing map is linear and readable, but zones need stronger purpose markers. | Entry, lens, color, mirror, vault, and garden each get a visible prop cluster and operator-readable label. |
| Reset Ops Reviewer | Can staff reset, monitor, hint, and repair without hidden knowledge? | Reset artifacts exist in text but need to stay visible throughout play. | Keep timer/reset card/key chute/clipboard in visual nodes and panel state. |
| Throughput Hint Captain | How does staff keep teams moving while preserving fun? | Hint states are command-accessible; stuck-state triggers need visible anchors. | Each beat gets a recognizable prop that hints can point to: detents, notches, edge labels, status lamps. |
| Simulation Evidence Analyst | What did the run prove vs. suggest? | Visual smoke proves density, not player comprehension. | Add plan and role findings now; next gate should run persona/team transcripts against the richer scene. |
| MUDDLE Playtest Designer | Can a player understand room/exits/commands without source code? | Commands are readable; visual hierarchy needs stronger differentiation. | Use larger styled renderer plus Prism-specific frames and more descriptive panel text. |
| MUDDLE Product Steward | Does this belong in shared UX or product host? | Palette support can be shared by frame names; room content belongs in AMAZE. | Add generic frame coloring to MUDDLE, keep Prism props/sequence in AMAZE. |

## Role-reviewed design changes approved for next slice

1. Distinct Prism palette and frame names in the visual scene.
2. Beam-state spine as the visible dependency graph.
3. Persistent operator/reset props as part of the room fiction.
4. More differentiated solved/unsolved prop states.
5. No new puzzle beats until the current six-beat chain feels compelling.
