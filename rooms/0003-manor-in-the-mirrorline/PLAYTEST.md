## Surprise log

| Surprise | Evidence | Decision |
|---|---|---|
| Deduction can become private faster than other room types. | Fighting-team harness pass highlights logic domination across P2-P5. | Make every category require a physical proof token or shared verification. |
| Spooky lighting is a design hazard, not just mood. | Accessibility-varied pass makes P4 a blocker until high-contrast path exists. | Design work-light path first, then add theatrical light. |
| The cabinet is the room's promise and biggest exploit. | Chaotic/fighting probes both point to selector guessing/force risk. | Add proof-locks or operator-visible proof checkpoints. |
| Physical outputs improve deduction quality. | Proof-token map gives every category a visible output and final home. | Treat each category solve as "earn token, then accuse," not just "know answer." |
| Token specificity matters. | Crest/seal/tile forms make the mapping easier to explain than generic proof tokens. | Use distinct silhouettes and tactile keys for each category. |
| Object behavior beats object marking. | P3 moved from "find mark" to "test object on balance plinth." | Prefer physical behavior tests over hidden symbols when a prop category is central. |
| Work-light first, atmosphere second. | P4 moved from shadow/reflection language to inspection windows and transparent overlays. | Required location solve must work in high-contrast inspection mode before spooky mood lighting. |
| Alibis must be handled, not read. | P2 moved from portrait text to truth-window token tests. | Suspect deduction should be a shared physical contradiction, not a private logic paragraph. |
| The finale needs gates, not just labels. | P5 moved from proof-token sockets plus selectors to proof-locked selector covers. | The cabinet should physically prevent unsupported accusation, not merely instruct players not to guess. |
| Space is part of the puzzle graph. | Proof tray and cabinet role lane make category state public before P5. | Layout must support injection/surjection too: every object needs a home that does not block flow. |
| Delight should be optional but physical. | House echoes add touchable/readable reward after proofs are earned. | Atmosphere should deepen the room without becoming homework or a hidden clue path. |
| Prototype order is a design tool. | The P0 sprint now tests flow, affordance, reset, force, and accessibility before scenic finish. | Build the cheapest evidence that can kill or promote the idea. |
| Content must be prototypeable too. | P0 now has a fixed original cast, object set, room set, and answer. | Names and labels should be tested for recall and originality before scenic art is commissioned. |
| The catalog works across room genres. | Manor and Brineworks now share technique/device/kit language despite different themes. | Keep adding catalog IDs when a room beat becomes build-serious. |
