# The Manor in the Mirrorline - Operations

## Staffing

- Operator count: one operator for simulation and prototype.
- Required training: intro/safety script, category-state tracking, final accusation protocol, manual reveal, reset checklist.
- Monitoring position: outside trailer or protected nook with views of door, accusation cabinet, and high-touch stations.

## Run clock

- Intro: 3-5 minutes.
- Soft-warning: 35 minutes.
- Hard-exit: 45 minutes.
- Reset: 10 minutes target.

## Operator category-state map

The operator should track categories as proof states, not just final answers. A
team is ready for the mirrorline cabinet only when each category has a visible
proof or an approved acceleration.

| Category | Player-facing proof | Operator-visible state | Acceleration | Blocks final cabinet? |
|---|---|---|---|---|
| Suspect | portrait-back truth window shows one alibi token breaking the mirrorline | suspect crest seated in cabinet or category called correctly | demonstrate one true alibi line and seat staff crest | yes |
| Object | sideboard object balances the mirrorline plinth and releases/identifies the object seal | object seal seated in cabinet or category called correctly | demonstrate one failed plinth test and seat staff seal | yes |
| Location | work-light inspection window shows which room refuses suspect/object overlay alignment | room-key tile seated in cabinet or category called correctly | demonstrate one false-room alignment and seat staff tile | yes |
| Final accusation | all three proof tokens are seated, selector covers open, then suspect/object/location are called, checked, and set | sockets, covers, and selector states visible to operator | seat staff token or open staff cover for proven categories | no, if staff bypass is needed |

Proof tokens should normally pause in the public proof rail before P5. If a team
keeps a proof token in a pocket or at a station, the operator should redirect it
to the rail before the final accusation.

## Hint protocol

| Beat/stuck state | Observable signal | Hint 1: point attention | Hint 2: clarify relationship | Acceleration |
|---|---|---|---|---|
| P1 category confusion | players search before naming categories | "The house ledger asks three questions." | "Guest, object, room: solve one at a time." | name the three categories |
| P2 suspect stall | rereading portraits only or not testing tokens | "The portraits are evidence, not biographies." | "Put each alibi token in its own truth window and watch the mirrorline." | demonstrate one true alibi line |
| P3 object chaos | objects scattered or visual-search handling | "The sideboard inventory matters." | "The sideboard is not asking you to stare; it is asking you to test." | demonstrate one failed balance test |
| P4 room station stall | no overlay/window comparison | "The manor inspection window needs both proofs." | "Use the suspect edge and object seal under the work light." | demonstrate one false-room alignment |
| P5 random accusation | selector-cover forcing, slider spinning, or missing proof tokens | "The manor rejects unsupported accusations." | "Seat proof, call it, check it, then set it." | seat staff proof token and open staff cover for proven categories |
| proof rail clutter | unrelated props or papers placed on proof rail | "Only proven answers belong on the rail." | "Crest, seal, and room-key only." | clear rail and name current proof state |
| optional echo confusion | team treats house echo as required clue | "That is the house remembering, not the ledger asking." | "Proof first; memories after." | cover echo until proof is earned |

## Reset timing budget

The first cardboard sprint should measure reset in minutes, not vibes.

| Reset segment | Target min | Stop/fix if over | Likely simplification |
|---|---:|---|---|
| Ledger and category plaques | 0.5 | 1.0 | fixed plaques instead of loose cards |
| Portrait truth windows and alibi tokens | 2.0 | 3.0 | keyed pockets, fewer loose tags, larger reset photo |
| Object tray and balance plinth | 1.5 | 2.5 | tether/oversize props or reduce object count |
| Room windows, overlays, and room-key tile | 2.0 | 3.0 | fewer room panels for prototype; larger overlays |
| Proof rail and house echoes | 1.0 | 1.5 | hide echoes by flap instead of loose cards |
| Cabinet sockets, covers, sliders, reveal | 2.0 | 3.0 | simplify cover mechanism before adding finish |
| Egress/loose-prop sweep | 1.0 | 1.5 | remove floor-adjacent storage |
| Total | 10.0 | 12.0 | cut optional echo/reset debt first, not safety checks |

## Failure modes

| Failure | Player-facing recovery | Staff action | Post-session fix |
|---|---|---|---|
| portrait frame sticks | "The housekeeper's copy is clearer." | give exclusion hint or open safely | inspect hinge/stop |
| alibi token missing, swapped, or unreadable | "The housekeeper keeps duplicate testimony." | provide duplicate token or demonstrate one true line | replace token and update reset map |
| object missing | "The sideboard inventory has a duplicate." | provide duplicate or mark card | replace/retether object |
| balance plinth sticks or misreads | "The sideboard mechanism is sticking; the steward can certify the object." | demonstrate intended result or seat staff seal | inspect pointer, rest surface, and true object weight |
| inspection overlay missing or scratched | "The housekeeper has a clean inspection plate." | provide spare overlay or printed alignment card | replace overlay |
| reveal/work light fails | "Use the housekeeper's daylight notes." | provide printed/high-contrast clue | repair LED/power |
| proof token missing or jammed | "The manor steward has the duplicate seal." | use staff proof token or bypass socket | replace token/check socket |
| proof rail becomes prop dump | "Only proven answers belong on the rail." | clear unrelated props and restore token homes | reduce rail surface or add silhouettes |
| echo card damaged or confusing | "The house memory is smudged; the proof is still valid." | remove/replace optional echo | rewrite shorter or hide until solved |
| selector cover sticks or opens without proof | "The manor steward will open the proven category." | open/hold cover manually after proof is shown | inspect cover stop/tab |
| cabinet selector slips | "The manor steward will hold that answer." | set/lock category manually | inspect knob/slider stop/set screw |
| final reveal fails | "The mirrorline has accepted the accusation." | release token manually | inspect cabinet/reveal |

## Operator stress tests

| Scenario | Expected staff action | Hard limit | Revision if failed |
|---|---|---|---|
| confused team searches everything | redirect to ledger categories | no aimless search past P1 hint time | stronger category signage |
| fighting team has one logician | assign caller/checker/handler | no player idle for whole scene | split evidence more physically |
| chaotic team guesses cabinet | pause covers/sliders and require proof | no repeated random final input | lock cabinet until proofs |
| accessibility-varied team cannot use inspection-window clue | work-light/high-contrast overlay path works | no low-light-only required clue | redesign P4 |
| late/tired group reaches hard-exit | accelerate one category and final reveal safely | no rushed unsafe cabinet use | simplify final sequence |

## Prototype run script

For the first cardboard run, the operator should not perform theatrical delivery.
The goal is evidence quality.

| Phase | Operator instruction | Evidence to capture |
|---|---|---|
| Before run | Photograph opening state, proof rail, cabinet covers, and egress. | reset photo set complete |
| During P2 | Watch whether players test alibi tokens before hint time. | first physical action timestamp |
| During P3 | Watch whether players use the plinth before searching for marks. | first plinth use timestamp |
| During P4 | Watch seated/low-light route under work light. | readable/not readable decision |
| During P5 | Assign or observe caller/checker/handler. | whether one player can dominate |
| After run | Time reset with one operator. | reset total and slowest segment |
