# Components

Reusable AMAZE inventory lives here. Room `BUILD.md` files should reference
inventory IDs instead of inventing one-off sourcing notes.

## Files

| File | Purpose |
|---|---|
| `INVENTORY.md` | Common parts, supplier classes, price bands, durability, spares, and substitution notes. |
| `SOURCING.md` | How AMAZE chooses common suppliers, validates prices, and avoids fragile one-off parts. |

## Inventory rule

Every promoted room component should answer:

1. Can we buy or fabricate it from a repeatable source?
2. What is the current price band?
3. What is the spare plan?
4. What is the acceptable substitute if the preferred item is unavailable?
5. What fails first under public use and transport?
